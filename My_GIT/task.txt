Разработать прототип системы контроля версий, которая хранит все данные локально в заданной директории 
(например, .<имя директории>/). Каждое изменение (коммит) имеет время жизни. 
По истечении этого времени коммит и связанные с ним данные должны быть автоматически удалены.

Консольное приложение, поддерживающее следующие команды: 
- init - Создает необходимые служебные файлы и папки
- push - Создает новый коммит, сохраняющий текущее состояние рабочей директории. 
Коммит должен иметь уникальный идентификатор, сообщение, время создания и время жизни коммита. 
Должны сохраняться только те файлы, которые изменились с момента предыдущего коммита
- pull - Восстанавливает состояние рабочей директории на момент указанного коммита
- log - Выводит список всех коммитов в читаемом формате. Для каждого коммита отображается: 
	ID, дата и время создания, время истечения срока действия, сообщение коммита.

Коммиты и объекты (файлы) должны храниться в поддиректориях (например, .<имя директории>/objects/a1b2c3d4...).
Для лога коммитов используйте отдельный файл (например в JSON формате).
Время создания коммита и его время жизни должны храниться в Unix timestamp.
Необходимо предусмотреть проверку срока действия при выполнении большинства команд.
